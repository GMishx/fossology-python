Fossology Jobs¶

Methods used to access “jobs/” endpoints.

class fossology.jobs.Jobs¶

Class dedicated to all “jobs” related endpoints

detail_job(job_id)¶

Get detailled information about a job

API Endpoint: GET /jobs/{id}

Parameters

job_id – the id of the job

Type

int

Returns

the job data

Return type

Job

Raises

FossologyApiError – if the REST call failed

list_jobs(page_size=20, pages=1)¶

Get all available jobs

API Endpoint: GET /jobs

The answer is limited to the first page of 20 results by default

Parameters

  • page_size (int (default to "20")) – the maximum number of results per page

  • pages (int (default to "1")) – the number of pages to be retrieved

Returns

the jobs data

Return type

list of Job

Raises

FossologyApiError – if the REST call failed

schedule_jobs(folder, upload, spec)¶

Schedule jobs for a specific upload

API Endpoint: POST /jobs

Job specifications spec are added to the request body, following options are available:

>>> {
>>>     "analysis": {
>>>         "bucket": True,
>>>         "copyright_email_author": True,
>>>         "ecc": True,
>>>         "keyword": True,
>>>         "monk": True,
>>>         "mime": True,
>>>         "monk": True,
>>>         "nomos": True,
>>>         "ojo": True,
>>>         "package": True
>>>     },
>>>     "decider": {
>>>         "nomos_monk": True,
>>>         "bulk_reused": True,
>>>         "new_scanner": True,
>>>         "ojo_decider": True
>>>     },
>>>     "reuse": {
>>>         "reuse_upload": 0,
>>>         "reuse_group": 0,
>>>         "reuse_main": True,
>>>         "reuse_enhanced": True
>>>     }
>>> }
Parameters

  • folder (Folder) – the upload folder

  • upload (Upload) – the upload for which jobs will be scheduled

  • spec (dict) – the job specification

Returns

the job id

Return type

Job

Raises

FossologyApiError – if the REST call failed
